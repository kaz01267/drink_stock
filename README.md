# DrinkStock

## サービス概要

DrinkStockは、家にあるお酒の在庫管理と飲酒記録を一元化できる個人向けWebアプリです。記録データをもとに、今日飲むお酒の提案や直近の飲酒状況の可視化を行い、無理なく継続して使える体験を提供します。将来的には、感想や思い出も残せるようにし、お酒にまつわる体験を振り返れるサービスを目指します。


## このサービスへの思い・作りたい理由

家にお酒の在庫が増えてくると、「何がどれくらい残っているのか分からない」、「結局いつも同じお酒を選んでしまう」といった状況になりがちでした。
また、自宅で飲んだお酒や、居酒屋や旅行先で飲んだお酒について、感想やその時の出来事を振り返りたいと思っても、
後から見返せる形で記録が残っていないことに不便さを感じていました。
さらに、飲酒後は記録を忘れてしまうことも多く、継続して記録すること自体が難しいと感じていました。
そこで、在庫管理と飲酒記録を一元化し、「今日は何を飲もうか」と考える時間を提案機能によってサポートするとともに、
簡単に記録を残せて、後から振り返ることに意味を持たせた、自分専用の飲酒管理・記録サービスを作りたいと考え、本サービスを企画しました。

## ユーザー層について

- 自宅でお酒を飲むことが多い人

- 複数種類のお酒をストックしている人

- お酒の記録や思い出を残したい人

- 在庫管理と飲酒記録をシンプルに行いたい個人ユーザーを主な対象としています。

## サービスの利用イメージ

在庫として登録されているお酒は、一覧画面や「今日のお酒」提案画面からワンタップで飲酒記録を作成できます。

飲酒記録に応じて在庫数量は自動的に更新され、ユーザーは現在の在庫状況や直近の飲酒状況を把握で切るようにし
直近1週間の飲酒回数や量などを見えるようにすることで自分の飲酒ペースを振り返ることも可能です。
また、在庫情報と直近の飲酒履歴をもとに「今日のお酒」を提案する機能を提供します。
毎日決まった時間に提案内容を通知されるように設定できます。
さらに、居酒屋や旅行先などで飲んだお酒については在庫とは連動しない「お酒ログ」として、感想や思い出を自由に記録できるページも実装予定です。
このように「何を飲むか提案がくる」、「簡単に記録ができる」、「後から振り返れる」ようにして自分専用の記録サービスを目指しています。

## サービスの差別化ポイント・推しポイント

DrinkStockは、在庫管理と飲酒記録を一体化しており、
「飲んだら記録する」だけで在庫状況が自動的に更新される点を特徴としています。
これにより、現在の在庫を意識しながら、無理なく飲酒記録を残すことができます。

また、個人利用に特化し、通知やワンタップ記録などを取り入れることで、
飲酒後でも負担なく記録でき、継続して使いやすいシンプルなUIを重視しています。

飲酒量や頻度を直近期間で可視化することで、
自分の飲酒傾向を把握でき、飲み過ぎ防止や振り返りにつなげられる点も特徴です。

既存の飲酒管理アプリや記録アプリが「健康管理」や「ログの蓄積」に特化しているのに対し、
DrinkStockは「何を飲むか考える → 飲む → 記録する → 振り返る」という
日常的な一連の行動を一つのサービスで完結できる点に価値を置いています。

提案機能・記録・振り返りを通じて、
単なる在庫管理に留まらず「お酒にまつわる体験」そのものを残し、
後から楽しめるサービスであることを推しポイントとしています。

## 機能候補

### MVPリリース時に作りたい機能（最小構成）

- ユーザー登録 / ログイン

- お酒の在庫登録・編集・削除  
  （家にあるお酒を事前に登録できる）

- 飲酒記録の登録  
  （登録済みのお酒を選択するだけで簡単に記録できる）

- 飲酒記録に基づく在庫数量の自動更新  
  （飲んだ分が即時反映され、現在の在庫状況が分かる）

### 本リリース・将来的に追加したい機能

- 「今日のお酒」提案機能  
  （在庫状況や直近の飲酒履歴をもとにおすすめを表示）

- 提案通知機能  
  （提案が出たタイミングで通知し、サービスを思い出すきっかけを作る）

- 飲酒量・頻度の可視化  
  （直近1週間などを目安に、どれくらい飲んだかを振り返られる）

- 飲んだお酒への感想・思い出の記録  
  （居酒屋や旅行先など、過去に飲んだお酒も含めて自由に記録）

- 飲酒履歴・思い出の振り返り機能  
  （時系列で見返し、「どんなお酒をどう楽しんできたか」を確認できる）
